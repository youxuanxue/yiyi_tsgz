# 哟哟MBTI - 动态性格测试H5应用

一款创新的H5 Web应用，通过模拟社交互动，实时、动态地展示和分析用户的MBTI性格倾向。

## 🎯 核心特性

- **实时动态MBTI**：性格不再是静态标签，而是根据行为实时变化
- **每日社交挑战**：通过热点新闻和生活情境的选择来探索性格倾向
- **八维度可视化**：E、I、S、N、T、F、J、P 八个独立维度的实时展示
- **双向坐标轴**：直观展示维度对比，标签位于坐标轴两端
- **每周统计报告**：自动统计本周主要类型和类型分布
- **类型分布散点图**：16种MBTI类型的可视化分布，占比大的点大，占比小的点小
- **性格维度柱状图**：4组维度对的柱状图展示，同一组使用相同颜色
- **每周回归调整**：防止得分无限累积，体现长期变化

## 🛠️ 技术栈

- **Vue 3** - 渐进式JavaScript框架
- **Vite** - 下一代前端构建工具

- **Canvas API** - 图表绘制

- 展示用户形象和当前MBTI类型
- 八维度双向坐标轴可视化
  - 左侧显示：I、N、F、P（红色系）

## 🛠️ 核心机制

### 维度计分模型
- 八个独立维度：E、I、S、N、T、F、J、P
- 分值范围：0 到 100
- 每个维度独立计分，互不影响
- 通过维度对比确定MBTI类型：
  - E vs I：E得分 > I得分 → E，否则 → I
  - S vs N：S得分 > N得分 → S，否则 → N
  - T vs F：T得分 > F得分 → T，否则 → F
  - J vs P：J得分 > P得分 → J，否则 → P

### 选择权重分配
每个挑战选项都有明确的维度倾向和权重：
- 维度：直接指定影响的维度（E、I、S、N、T、F、J、P）
- 权重：影响程度（15-20分）
- 机制：选择时增加对应维度得分，同时减少对立维度得分

### 类型切换规则
- 实时切换：当维度对比发生变化时（例如E得分超过I得分），立即切换对应字母
- 每周回归：每周自动微调8%回归初始值
